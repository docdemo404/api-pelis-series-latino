🎬 API Películas & Series Latino

Documentación oficial humana y machine-readable para desarrolladores y Agentes de IA (LLMs).

🤖 Especificación OpenAPI 3.0 (JSON)
✨ Catálogo Unificado & Gestor de Fuentes: La API agrupa automáticamente títulos idénticos entre diferentes proveedores (TioPlus, FuegoCine, Supabase) unificando sus servidores. Puedes gestionar la activación y prioridad de fuentes en /panel o /api/v1/panel.
🎬 TMDB API Enriquecida: Integración con la API oficial v3 de TheMovieDatabase (TMDB). Todos los contenidos devuelven sinopsis oficiales en Latino, tráilers oficiales en YouTube, pósters/backdrops en alta definición, reparto con fotos y calificaciones.
🤖 Machine-Readable API & Alta Velocidad (Edge CDN & In-Memory Cache): Respuestas ultrarrápidas de sub-20ms con Edge Caching (Cache-Control: public, s-maxage=86400, stale-while-revalidate), soporte de batching (/api/v1/media/batch?ids=...), paginación real sobre el índice general (?page=2&limit=25 o ?page=2&size=25) y proyecciones compactas (?compact=true). Especificación OpenAPI 3.0 disponible en /api/v1/openapi.json.
⚠️ Regla de Consumo de Servidores: El campo status de cada servidor en servers indica su disponibilidad real comprobada en capa de aplicación (inspección de Soft Errors). Todo reproductor o aplicación cliente DEBE IGNORAR O FILTRAR los servidores que retornen "status": "offline".

📌 Endpoints Principales

GET / POST /panel | /api/v1/panel

Panel de Gestión de Fuentes: Interfaz interactiva y endpoint JSON para activar/desactivar proveedores de scraping (TioPlus, FuegoCine, Supabase DB) y reordenar su prioridad de visualización.

GET /api/v1/search?q=avatar

Búsqueda General: Busca todo tipo de contenidos (Películas, Series, Animes y Doramas) sobre el catálogo indexado con coincidencia por substring (ej. ma encuentra Matilda). El índice incluye el título original y los alias de cada ficha, así que vengadores encuentra Avengers 2: Era de Ultrón y home alone encuentra Mi pobre angelito. Respuesta lean por defecto (sin cast ni reproductores) para máxima velocidad — el detalle se obtiene en /api/v1/media/{id}; usa ?full=true para el objeto completo. Reporta total_results como el total real del catálogo, con has_more y next_page para scroll infinito.

GET /api/v1/openapi.json

Especificación completa OpenAPI 3.0.3 en formato JSON para lectura de máquinas, validación de esquemas y generación de clientes.

GET /api/v1/feeds/home?country=CL

Catálogo de inicio: hero rotatorio (spotlight, hasta 5 títulos con ficha completa: logo, sinopsis, duración, clasificación, reparto y tráiler) y alrededor de 15 carruseles temáticos — Top 10 del país, novedades, películas, series, mejor valoradas, anime, acción, comedia, terror, ciencia ficción, animación, drama, romance, aventura, crimen, documentales y clásicos. Una fila se omite si no reúne al menos 8 títulos, de modo que nunca se publica un carrusel a medias.

Parámetros: ?detail=card|compact (metadata completa o payload mínimo), ?rows=action,anime (carga por lotes) y ?limit=20 (ítems por carrusel, máx. 40). Cada fila trae layout (ranked, backdrop, poster) como pista de render y endpoint para su "ver todo" paginado.

GET /api/v1/media/scary-movie-6

Ficha instantánea: devuelve toda la metadata (sinopsis, imágenes, logo, duración, clasificación, reparto y temporadas) desde la base de datos o la caché, sin esperar a la resolución de enlaces. El bloque streams indica si los servidores ya están listos ("status": "ready") o dónde pedirlos ("status": "pending").

Usa ?streams=wait si necesitas metadata y servidores en una sola respuesta (comportamiento clásico, más lento).

GET /api/v1/media/{id}/streams

Enlaces de reproducción: resuelve los servidores del título y los persiste para las siguientes consultas. Pensado para llamarse al pulsar Reproducir, con la ficha ya pintada. ?deep=1 fuerza la fusión multifuente completa (TioPlus + FuegoCine). Equivalente para series: /api/v1/series/{id}/streams.

⚡ Flujo recomendado para fichas emergentes: las tarjetas del home ya incluyen la metadata completa, así que el popup se puede pintar al instante con los datos que ya tienes. Llama a /api/v1/media/{id}/streams solo cuando el usuario decida reproducir. Si necesitas abrir la ficha desde una búsqueda (payload lean), pide /api/v1/media/{id}: responde sin scraping y te dice en streams si los enlaces ya están resueltos.
